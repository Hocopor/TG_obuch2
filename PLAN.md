## Журнал решений

- **2025-07-15 — Стек**: Python + aiogram 3 (бот), FastAPI + Jinja2 + SQLAlchemy (админка), PostgreSQL (БД), Docker (контейнеры), Caddy (reverse proxy). Обоснование: один язык, минимальная сложность, легко поддерживать.
- **2025-07-15 — БД**: PostgreSQL в Docker-контейнере. Connection pooling через SQLAlchemy (asyncpg). Общий доступ бота и админки к одной БД.
- **2025-07-15 — Порты**: бот — фоновый процесс (не слушает порт), админка — 5000, PostgreSQL — 5433 (внутренний). Диапазоны 3000-4000 и 8000-9000 свободны.
- **2025-07-15 — Caddy**: домен `andrew-bot-adminka.mak-o.ru` уже направлен на сервер. Caddy проксирует на порт 5000.
- **2025-07-15 — Файлы**: тестовые изображения и видео переименованы в `files/`. Реальные файлы будут загружаться через админку.
- **2025-07-15 — Оплата**: на стороне GetCourse. Бот только фиксирует факт покупки. В будущем — интеграция с GetCourse API для проверки покупок и управления доступами.
- **2025-07-15 — Юр-документы**: загружаются через админку (файлы), хранятся на диске, отдаются пользователю по ссылке.
- **2025-07-15 — Инфраструктура**: docker-compose.yml создан, Dockerfile-ы для bot (python:3.12-slim + aiogram) и admin (python:3.12-slim + uvicorn), requirements.txt с зафиксированными версиями. shared/config.py читает .env, shared/database.py — async engine + sessionmaker, shared/models.py — 8 таблиц с relationship-ами.
